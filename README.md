## Folder Descriptions

### `fonts/` [AIO-FONT]
**Aioverse Code**: `AIO-FONT`  
**Category**: Typography System  
**Purpose**: Complete typography framework for all brand communications  

Contains typeface files organized by usage weight and purpose:
- `primary/Nohemi/` - Primary typeface for headlines and primary content
- `secondary/TokyoTrailMono/` - Accent typeface for technical/code contexts

**Usage**: Load fonts into design tools, web projects, and digital products  
**Naming**: `[FontName]-[Weight]-[Style].[format]`

---

### `logos/` [AIO-LOGO]
**Aioverse Code**: `AIO-LOGO`  
**Category**: Brand Marks  
**Purpose**: Primary and alternate brand identities  

Official Aioverse logo marks in multiple formats and color variations

**Usage**: Official brand representation across all media  
**Naming**: `Aioverse-[Type]-[Version]-[ColorMode].[format]`

---

### `icons/` [AIO-ICON]
**Aioverse Code**: `AIO-ICON`  
**Category**: UI Symbols & Icons  
**Purpose**: Comprehensive icon system for user interfaces  

**Usage**: UI development, app interfaces, documentation  
**Naming**: `[IconName]-[Size]-[State].[format]`

---

### `illustrations/` [AIO-ILLUS]
**Aioverse Code**: `AIO-ILLUS`  
**Category**: Custom Artwork & Visual Narratives  
**Purpose**: Brand illustrations for marketing, UI, and documentation  

**Usage**: Web pages, product demos, marketing materials, documentation  
**Naming**: `[Concept]-[Scene]-[Version].[format]`

---

### `photos/` [AIO-PHOTO]
**Aioverse Code**: `AIO-PHOTO`  
**Category**: Brand Photography & Imagery  
**Purpose**: Curated branded photographs for visual communications  

Professional photography organized by subject matter

**Usage**: Websites, marketing campaigns, social media, presentations  
**Naming**: `Aioverse-[Subject]-[DateTaken]-[Reference].[format]`

---

### `guidelines/` [AIO-GUIDE]
**Aioverse Code**: `AIO-GUIDE`  
**Category**: Brand Standards & Documentation  
**Purpose**: Brand guidelines and usage specifications  

Comprehensive brand guidelines for consistent implementation

**Usage**: Brand training, design resources, developer reference  
**Naming**: `Aioverse-[Aspect]-Guidelines-[Version].[format]`

---

### `tokens/` [AIO-TOKEN]
**Aioverse Code**: `AIO-TOKEN`  
**Category**: Design System Variables & Tokens  
**Purpose**: Programmatic design tokens for consistency  

Design system tokens in multiple export formats (JSON, CSS, YAML)

**Usage**: Design tools, CSS variables, component libraries, design systems  
**Naming**: `aioverse-[domain]-[version].[format]`

## Naming Convention

All assets follow: `[AssetType]-[Descriptor]-[Variant]-[Version].[Format]`

Examples:
- `Aioverse-Logo-Primary-Dark-v1.svg`
- `Icon-Menu-24-Active.svg`
- `Photo-Team-20250110-v1.jpg`

**For complete guidelines**, see **[AIOVERSE_NAMING.md](./AIOVERSE_NAMING.md)**

## Quick Reference

### For Designers
1. Download fonts from `fonts/` folder
2. Reference logo files from `logos/` folder
3. Check `guidelines/` for brand standards
4. Use icons and illustrations from respective folders

### For Developers
1. Import design tokens from `tokens/` for CSS variables
2. Use SVG icons from `icons/` folder
3. Reference color and typography values
4. Follow Aioverse naming conventions
